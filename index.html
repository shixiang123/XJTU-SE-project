Abstract

XJTUhub is an XJTU-oriented campus assistant system that combines a WeChat mini-program frontend and a Node.js backend service. The frontend covers high-frequency student scenarios including login, weekly schedule, term score query, resource plaza, attendance panel, and DDL management. The backend provides unified authentication and crawler-triggered data acquisition from eHall-related systems, then maps heterogeneous raw data into stable API contracts for the mini-program. The project emphasizes practical usability, data-structure compatibility, and maintainable school-provider extension for future multi-school support.

Core Features

Authentication

  • Student ID + password login
  • Captcha-based login initialization and verification
  • Token-based session handling with auto redirect on invalid auth

Course Schedule

  • Weekly timetable rendering
  • Current-week auto calculation
  • Course detail pop-up (teacher, location, week range, sections, credit)

Score Query

  • Term-based score list switching
  • Raw score breakdown (normal/midterm/final/composite)
  • Local cache + manual refresh

Learning Assistant

  • Resource plaza with category filter and keyword search
  • Attendance and DDL modules (current local demo datasets)
  • Function aggregation page + customized TabBar UX

System Architecture

Frontend (XJTUhub-main)

Built on native WeChat mini-program framework with Vant Weapp and ColorUI. It uses a unified request wrapper for token injection, error-code interpretation, and login fallback. Main pages include login, index resources, course, score, attendance, DDL, and mine.

Backend (XJTU-API-main)

Node.js service exposing stable REST endpoints for the mini-program. It supports login token issuance, crawler trigger/status query, and mapped output endpoints for courses and scores. Crawler outputs are normalized and persisted to JSON artifacts for retrieval and debugging.

Implementation Snapshot

Repository structure in final branch:

  • XJTUhub-main/: WeChat mini-program frontend
  • XJTU-API-main/: Node.js backend + crawler scripts
  • docs/backend-realtime-crawler-spec.md: backend compatibility and extension specification

Current integrated API set:

  • POST /login, GET /login-init, POST /login-verify, GET /login-code
  • GET /courses, GET /scores, GET /raw-scores
  • Backend crawler-related: POST /crawl/xjtu/trigger, GET /crawl/xjtu/status

Response contract: { code, msg, data } with code=0/-1/403 semantics for frontend compatibility.
